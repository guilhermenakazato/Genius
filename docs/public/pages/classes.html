Organização

A organização do projeto Genius é feito em duas pastas: backend e frontend. Na frontend, as pastas mais importantes são a lib (onde todas as classes Flutter ficam), a assets (onde todas as fotos são configuradas) e a fonts (onde todas as fontes ficam), enquanto que na backend as pastas mais importantes são start (onde as rotas de acesso ao BD são configuradas), database (onde o banco de dados é configurado) e app (onde as ações executadas em cada rota são configuradas).

É importante ressaltar que dentro da pasta lib há 5 pastas importantes: components, http, models, screens e utils. Na components, são armazenados componentes reutilizáveis; na http, são colocados códigos referentes à configuração de requisições HTTP dentro do app Flutter; na models, são guardados os models, classes que representam uma tabela de dados; na screens, estão todos os códigos referentes às telas do Genius e, por fim, na utils, são colocados métodos que podem vir a ser úteis durante o desenvolvimento. Dentro da pasta screens, há 2 sub-pastas importantes: signup e main. Na signup, está todo o código referente ao cadastro do usuário, enquanto que na main está todo o código relacionado à tela principal do Genius, onde se situa o Feed. Dentro da pasta main, há 4 sub-pastas: project, follows, edit e user_info. Na edit, fica todo o código referente à edição de perfil do usuário, ou seja, informações do usuário, questionários do usuário, projetos do usuário e conquistas do usuário. Na follows, fica o código que constrói todo o layout das telas de seguidores. Na project, é armazenado o código referente à criação de projetos dentro do app, além do código de mostrar as informações do app ao clicar no Card do Feed. E, por, fim, na pasta user_info, fica todo o código que constrói o layout das informações que aparecem no perfil, como o "Sobre mim" e "Meus projetos".

Backend

Classe: AchievementsController

Caminho: backend\app\Controllers\Http\AchievementsController.ts

Classe responsável por receber as requisições referentes às conquistas.

Métodos da classe:

  • createAchievement: recebe uma conquista pelo método POST e a cria. Para mais informações sobre os dados de uma conquista, olhe o model Achievement.
  • listAllAchievements: lista todas as conquistas existentes no BD.
  • getAchievementById: recebe o id de uma conquista. Se ela existir no BD, retorna seus dados.
  • updateAchievement: recebe o id de uma conquista e os dados a serem atualizados. Se a conquista existir no BD, retorna os dados já atualizados.
  • deleteAchievement: recebe o id de uma conquista. Se a conquista existir no BD, será excluída.

Classe: AuthController

Caminho: backend\app\Controllers\Http\AuthController.ts

Classe responsável por receber as requisições referentes à autenticação do usuário.

Métodos da classe:

  • login: recebe o email e a senha do usuário. Se os dados estiverem corretos, retorna um token de autenticação.
  • authenticateWithToken: recebe o token de autenticação disponibilizado pelo método login e verifica se está correto. Se estiver, autentica o usuário com sucesso.
  • logout: desloga o usuário e desvalida o token da sessão.
  • getUserData: verifica se o token é válido. Se for válido, retorna os dados do usuário logado.
  • checkTokenIsValid: recebe um token e verifica se ele é válido ou não, retornando um bool de acordo com o resultado.

Classe: ProjectsController

Caminho: backend\app\Controllers\Http\ProjectsController.ts

Classe responsável por receber as requisições referentes aos projetos.

Métodos da classe:

  • createProject: recebe um projeto e o cria. Para mais informações sobre os dados de um projeto, olhar o model Projeto.
  • updateProject: recebe o id de um projeto e os dados a serem atualizados. Se o projeto existir no BD, retorna os dados já atualizados.
  • updateDeleteRequests: recebe o id de um usuário e o id de um projeto. Serve para verificar se um projeto deve ser deletado ou não, sendo isso feito quando as requisições de exclusão de projeto são iguais à quantidade de participantes do mesmo. Ex: 3 votos e 3 participantes = exclusão.
  • listAllProjects: lista todos os projetos existentes na tabela de dados.
  • getProjectById: recebe o id de um projeto através de parâmetros HTTP. Caso o projeto exista, é retornado como dado.
  • verifyIfProjectTitleAlreadyExists: recebe o nome de um projeto através de uma requisição POST e o retorna caso exista.
  • allParticipantsExist: verifica se todos os participantes de um array passado na função existem, sendo usado na atualização e criação de projetos.
  • createProjectTagRelationship: responsável por associar as tags passadas pela requisição POST a um projeto.
  • createProjectParticipantRelationship: responsável por associar os participantes passadas pela requisição POST a um projeto.
  • updateProjectTagRelationship: responsável por atualizar as tags de um projeto, excluindo-as ou adicionando-as.
  • updateProjectParticipantRelationship: responsável por atualizar os participantes de um projeto, excluindo-os ou adicionando-as.

Classe: UsersController

Caminho: backend\app\Controllers\Http\UsersController.ts

Classe responsável por receber as requisições referentes aos usuários.

Métodos da classe:

  • listAllUsers: lista todos os usuários existentes na tabela de dados.
  • createUser: recebe um usuário e o cria. Para mais informações sobre os dados de um usuário, olhar o model Usuário.
  • getUserById: recebe o id de um usuário. Se o mesmo existir na tabela de dados, retorna seus dados.
  • deleteUser: recebe o id de um usuário. Se o mesmo existir na tabela de dados, deleta-o.
  • updateUser: recebe o id de um usuário e os dados a serem atualizados. Se o usuário existir no BD, retorna os dados já atualizados.
  • saveProject: recebe o id de um projeto e o id de um usuário através de uma requisição POST. Caso ambos existam, o projeto fica salvo nos dados do usuário como sendo um de seus favoritos.
  • verifyIfUsernameAlreadyExists: recebe o nome do usuário e verifica se o mesmo já existe, sendo isso utilizado na criação e atualização de usuários.
  • verifyIfEmailAlreadyExists: recebe o email de um usuário e verifica se o mesmo já está em uso, sendo isso utilizado na criação e atualização de usuários.
  • updateUserTagRelationship: responsável por atualizar as tags favoritas de um usuário, excluindo-as ou adicionando-as.

Classe: Achievement

Caminho: backend\app\Models\Achievement.ts

Classe responsável por representar os campos do banco de dados referentes à tabela de conquistas.

Variáveis da classe:

  • id: variável númerica e única que serve para identificar qual é o projeto.
  • user_id: variável númerica que serve para identificar a quem pertence a conquista.
  • institution: variável do tipo texto que serve para indicar aonde a conquista foi realizada.
  • name: variável do tipo texto que serve para dar um nome à conquista.
  • position: variável opcional do tipo texto que serve para mostrar a posição conquistada.
  • type: variável do tipo texto que serve para indicar se a conquista é uma medalha, honra ao mérito, certificado ou outro tipo.
  • customizedType: variável opcional do tipo texto que serve para indicar qual seria o outro tipo, caso o tipo selecionado no campo "type" seja "Outro".

Classe: Project

Caminho: backend\app\Models\Project.ts

Classe responsável por representar os campos do banco de dados referentes à tabela de projetos.

Variáveis da classe:

  • id: variável númerica e única que serve para identificar qual é o projeto.

  • start_date: variável do tipo texto que serve para armazenar quando que o projeto se iniciou.
  • participants: variável do tipo lista que serve para indicar os participantes do projeto, incluindo orientador e coorientador.
  • savedBy: variável do tipo lista que serve para indicar quem salvou o projetos nos favoritos.
  • tags: variável do tipo lista que serve para indicar as tags associadas a um projeto.
  • deleteRequests: variável do tipo lista que serve para indicar quantas pessoas votaram para deletar o projeto.
