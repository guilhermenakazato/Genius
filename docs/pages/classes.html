Organização

A organização do projeto Genius é feito em duas pastas: backend e frontend. Na frontend, as pastas mais importantes são a lib (onde todas as classes Flutter ficam) e a assets (onde todas as fotos são configuradas), enquanto que na backend as pastas mais importantes são start (onde as rotas são configuradas), database (onde o banco de dados SQL é configurado) e app (onde as ações executadas em cada rota são configuradas).

É importante ressaltar que dentro da pasta lib há 5 pastas importantes: components, http, models, screens e utils. Na components, são armazenados componentes reutilizáveis; na http, são colocados códigos referentes às requisições HTTP dentro do app Flutter; na models, são guardados os models, classes que representam uma tabela de dados; na screens, estão todos os códigos referentes às telas do Genius e, por fim, na utils, são colocados métodos que podem vir a ser úteis durante o desenvolvimento.

Backend

Classe: AuthController

Caminho: backend\app\Controllers\Http\AuthController.ts

Classe responsável por receber as requisições referentes à autenticação do usuário.

Métodos da classe:

  • login: recebe o email e a senha do usuário. Se os dados estiverem corretos, retorna um token de autenticação.
  • token: recebe o token de autenticação disponibilizado pelo método login e verifica se está correto. Se estiver, autentica o usuário com sucesso.

Classe: ProjetosController

Caminho: backend\app\Controllers\Http\ProjetosController.ts

Classe responsável por receber as requisições referentes aos projetos.

Métodos da classe:

  • create: recebe um projeto e o cria. Para mais informações sobre os dados de um projeto, olhar o model Projeto.
  • index: lista todos os projetos existentes na tabela de dados.

Classe: UsuariosController

Caminho: backend\app\Controllers\Http\UsuariosController.ts

Classe responsável por receber as requisições referentes aos usuários.

Métodos da classe:

  • index: lista todos os usuários existentes na tabela de dados.
  • create: recebe um usuário e o cria. Para mais informações sobre os dados de um usuário, olhar o model Usuário.
  • get: recebe o id de um usuário. Se o mesmo existir na tabela de dados, retorna seus dados.
  • getId: recebe o email de um usuário. Se o mesmo existir na tabela de dados, retorna o id do usuário.

Classe: Projeto

Caminho: backend\app\Models\Projeto.ts

Classe responsável por representar os campos do banco de dados referentes à tabela de projetos.

Variáveis da classe:

  • id: variável númerica e única que serve para identificar qual é o projeto.
  • name: variável do tipo texto que serve para armazenar o nome do projeto.
  • tag: variável do tipo texto que serve para armazenar o tipo do projeto (tecnologia, ciências da natureza, etc).
  • orientador: variável do tipo texto que serve para armazenar o nome do orientador do projeto.
  • coorientador: variável não obrigatória do tipo texto que serve para armazenar o nome do coorientador.
  • qtde_orientandos: variável númerica que serve para armazenar quantos integrantes têm no projeto.
  • instituicao: variável do tipo texto que serve para armazenar aonde os integrantes trabalham no projeto.
  • data_inicio: variável do tipo texto que serve para armazenar quando que o projeto se iniciou.

Classe: Usuario

Caminho: backend\app\Models\Usuario.ts

Classe responsável por representar os campos do banco de dados referentes à tabela de usuários.

Variáveis da classe:

  • id: variável númerica e única que serve para identificar qual é o usuário.
  • username: variável do tipo texto que serve para armazenar o nome do usuário.
  • email: variável do tipo texto que serve para armazenar o email do usuário.
  • password: variável do tipo texto que serve para armazenar a senha do usuário.
  • type: variável do tipo texto que serve para armazenar o tipo do usuário (estudante, professor ou comunidade externa).
  • age: variável do tipo texto que serve para armazenar a idade do usuário.
  • local: variável do tipo texto que serve para armazenar aonde o usuário mora.

Classe: Routes

Caminho: backend\start\routes.ts

Arquivo responsável por definir as rotas de requisição HTTP.

Rotas:

  • "/usuarios": chama a classe UsuariosController e o método index.
  • "/usuarios/:id": chama a classe UsuariosController e o método get.
  • "/usuarios/id/:email": chama a classe UsuariosController e o método getId.
  • "/usuario": chama a classe UsuariosController e o método create.:
  • "/login": chama a classe AuthController e o método login.
  • "/token": chama a classe AuthController e o método token.
  • "/projeto": chama a classe ProjetosController e o método create.
  • "/projetos": chama a classe ProjetosController e o método index.

Frontend

Classe: ButtonRow

Caminho: frontend\lib\components\button_row.dart

Classe responsável por armazenar um componente com 2 botões que oferecem as opções sim e não.

Métodos da classe:

  • build: constrói um Widget (ButtonRow).

Variáveis da classe:

  • simScreen: recebe uma tela. Quando o usuário clicar no botão sim, o app entra nessa tela.
  • naoScreen: recebe uma tela. Quando o usuário clicar no botão não, o app entra nessa tela.

Classe: Button

Caminho: frontend\lib\components\button.dart

Classe responsável por armazenar um botão.

Métodos da classe:

  • build: constrói um Widget (Button).

Variáveis da classe:

  • text: recebe um texto e o mostra dentro do botão.
  • onClick: recebe uma função que é executada quando o botão é clicado.
  • width: recebe um valor double que determina a largura do botão.
  • height: recebe um valor double que determina a altura do botão.

Classe: Input

Caminho: frontend\lib\components\input.dart

Classe responsável por armazenar um componente que recebe texto escrito pelo usuário.

Métodos da classe:

  • createState: constrói um Widget de acordo com o State retornado (Input)

Variáveis da classe:

  • hint: recebe um texto que serve de dica para o que deve ser inserido no Input.
  • obscure: recebe um valor boolean que determina se o texto deve ser oculto ou não.
  • controller: recebe um Controller que controla todo o texto inserido no Input.
  • _selected: variável local e privada da classe que determina se o Input foi clicado ou não.

Classe: LoggingInterceptors

Caminho: frontend\lib\http\interceptors\logging_interceptors.dart

Classe responsável por interceptar as requisições HTTP feitas no app.

Métodos da classe:

  • interceptRequest: recebe os dados da requisição do backend e os imprime.
  • interceptResponse: recebe os dados da resposta do backend e os imprime.

Classe: LoginWebClient

Caminho: frontend\lib\http\webclients\login_webclient.dart

Classe responsável por lidar com todas as requisições e respostas referentes ao login.

Métodos da classe:

  • login: recebe um objeto Auth e realiza uma requisição HTTP POST com os dados. Se for bem-sucedida, retorna um token de autenticação.
  • logged: recebe o token fornecido pelo método login e realiza uma requisição HTTP POST com o token. Se for bem-sucedida, retorna true, indicando que o usuário está logado no aplicativo.

Classe: WebClient

Caminho: frontend\lib\http\webclient.dart

Classe responsável pelas configurações iniciais das requisições.

Variáveis da classe:

Classe: Login

Caminho: frontend\lib\screens\login.dart

Classe responsável por realizar o login do usuário.

Métodos da classe:

  • createState: constrói um Widget de acordo com o State retornado (Login)
  • build: constrói um Widget (Cadastro).
  • showSnackBar: recebe uma variável do tipo texto e o contexto e imprime o texto em uma SnackBar.

Sub-classes importantes:

  • _LoginStateBody: responsável por controlar o ProgressHUD.
  • _LoginStateContent: responsável por controlar a SnackBar e mostrar todo o conteúdo da tela de Login.

Variáveis da classe:

  • _emailController: instância do objeto Controller que controla todos os dados inseridos no input de email.
  • _passwordController: instância do objeto Controller que controla todos os dados inseridos no input de email.
  • progressContext: contexto da tela (é usado no ProgressHUD).
  • email: variável do tipo texto que recebe o texto do Controller de email.
  • senha: variável do tipo texto que recebe o texto do Controller de senha.
  • _webClient: instância do webClient, monitorando todas as requisições e respostas do backend.
  • navigator: instância do navigatorUtil, serve para navegar de uma tela a outra.
  • progress: instância do ProgressHUD, usado para manusear as funções da classe.
  • token: instância da classe Token, servindo para pegar o token presente no backend.
  • logged: variável do tipo bool que serve para verificar se o usuário já está logado ou não.
